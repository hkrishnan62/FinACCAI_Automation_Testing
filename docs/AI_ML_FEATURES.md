# AI/ML Features in FinACCAI

## Overview

FinACCAI uses **multiple AI and Machine Learning techniques** to enhance accessibility testing beyond traditional rule-based checks. The system combines:

1. **Machine Learning (ML)** - Pattern-based issue prediction
2. **Natural Language Processing (NLP)** - Text content analysis  
3. **Computer Vision** - Image accessibility analysis
4. **Explainable AI (XAI)** - Human-readable explanations

---

## ğŸ¤– Machine Learning Component

**File:** `finaccai/ml_model.py`

### What It Does:
- Extracts features from page structure (element counts, complexity metrics)
- Predicts potential accessibility issues based on patterns
- Provides confidence scores for predictions
- Identifies high-risk areas on complex pages

### Techniques Used:
- **Feature Engineering**: DOM structure analysis
- **Pattern Recognition**: Identifies common accessibility anti-patterns
- **Heuristic Models**: Currently uses intelligent heuristics (can be upgraded to Random Forest/Neural Networks)

### Example Predictions:
```python
{
    'type': 'high_image_count',
    'confidence': 0.85,
    'message': 'Page has many images - increased risk of missing alt text'
}
```

### Future Enhancements:
- Random Forest Classifier for multi-class prediction
- Neural Network for complex pattern detection
- Training on historical accessibility audit data
- Continuous learning from user corrections

## ğŸ“ Natural Language Processing (NLP)

**File:** `finaccai/nlp_analysis.py`

### What It Does:
- Analyzes text content for clarity and meaningfulness
- Evaluates form labels for descriptiveness
- Checks button and link text quality
- Detects vague or unclear instructions

### Techniques Used:
- **Text Analysis**: Keyword detection and pattern matching
- **Semantic Analysis**: Currently rule-based (can be upgraded to BERT/Transformers)
- **Readability Assessment**: Text length and complexity checks

### Example Findings:
```python

    "Button text 'click' could be more descriptive",
    "Non-descriptive link text: 'here'",
    "Very short label text: 'Go' - may not be descriptive enough"
]
```

### Future Enhancements:
- BERT-based text classification
- Transformer models for semantic understanding
- Readability scoring (Flesch-Kincaid, SMOG)
- Multi-language support
- Sentiment analysis for error messages

## ğŸ’¡ Explainable AI (XAI)

**File:** `finaccai/xai_explanations.py`

### What It Does:
- Generates human-readable explanations for findings
- Provides WCAG guideline references
- Assesses severity and impact
- Creates actionable recommendations

### Techniques Used:
- **Explanation Generation**: Context-aware reasoning
- **Impact Assessment**: User group analysis
- **Severity Classification**: Priority-based categorization
- **Recommendation Engine**: Actionable guidance

### Example Output:
```python
{
    'summary': 'Found 12 accessibility issues. Moderate attention needed.',
    'recommendations': [
        {
            'category': 'Images',
            'priority': 'High',
            'recommendation': 'Add descriptive alt text to 4 image(s)',
            'wcag': 'WCAG 2.1 Level A - 1.1.1 Non-text Content'
        }
    ],
    'severity_analysis': {
        'high': 8,
        'medium': 4,
        'low': 0
    },
    'impact_assessment': {
        'screen_reader_users': 'High - Missing alt text prevents understanding',
        'low_vision_users': 'High - Insufficient contrast makes text difficult'
    }
}
```

### Future Enhancements:
- SHAP (SHapley Additive exPlanations) for ML model interpretability
- LIME (Local Interpretable Model-agnostic Explanations)
- Grad-CAM for visual explanations of vision model decisions
- Attention visualization for NLP model decisions
